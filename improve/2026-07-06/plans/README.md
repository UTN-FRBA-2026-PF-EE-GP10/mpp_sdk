- `metrics.trap_depth` aliasing `final_efficiency`, and
  `tracking_efficiency` being the scalar case of `energy_efficiency`:
  intentional domain naming for the paper; consolidation would cost clarity.
- `METHODOLOGY_WARNING` banner printed by the static/dynamic harnesses:
  deliberate pedagogy, not stale text.
- `SpiMcuSource.read()` returning (0, 0) before the first `write()`:
  hardware-phase concern; revisit during HIL bring-up, not worth a plan now.
- Duplicated `_clamp` + constructor validation across the five algorithms
  and the identical restart-trigger block in ScanAndTrack/ParticleSwarm
  (~15 lines per copy): real but stable duplication; deferred to backlog to
  keep the algorithms self-contained for the MCU port. Reconsider when the
  next algorithm lands.
- Firmware PWM output on `PIN_25` (the on-board LED) instead of the board's
  gate-driver net on GPIO15, updated only every 100 ms: a known placeholder
  like the fake adc_task; becomes its own bring-up task after plan 005.
- Embassy git dependencies without a `rev=` pin: reproducibility is already
  covered by the committed `Cargo.lock`.
